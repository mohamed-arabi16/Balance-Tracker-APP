  <name>Task 1: EAS init, authenticate, and trigger production build</name>
  <files>
    BalanceTracker/app.json
  </files>
  <action>
    This task automates everything Claude can do. Steps that require user credentials are handled via interactive prompts — Claude will run the commands and let the user respond to auth prompts.

    **Step 1: EAS login**
    Run `eas login` from `BalanceTracker/`. This will prompt for Apple ID / EAS account credentials. If already logged in, it will show the current user.

    **Step 2: EAS init (link project)**
    Run `eas init` from `BalanceTracker/`. This creates an EAS project linked to the Expo account and writes the real `projectId` into `app.json`'s `extra.eas.projectId` field. If the project already exists in EAS dashboard, use `eas init --id <existing-project-id>`.

    **Step 3: Register bundle identifier (if needed)**
    Run `eas credentials` to check if iOS credentials exist. If no provisioning profile or distribution certificate exists, run `eas build --platform ios --profile production` — EAS will automatically create credentials (provisioning profile + distribution certificate) via Apple Developer Portal. This requires the Apple Developer Program enrollment to be active (see STATE.md blocker note).

    **Step 4: Trigger production build**
    Run: `cd BalanceTracker && eas build --platform ios --profile production`

    This will:
    - Upload the project to EAS Build servers
    - Compile a Release .ipa with the production signing identity
    - Upload the .ipa to TestFlight automatically (EAS handles this with `--auto-submit` flag OR the user can submit separately)

    Use: `eas build --platform ios --profile production --non-interactive` to avoid interactive prompts where possible.

    The build runs on EAS servers (cloud) — Claude does NOT need to wait for it to complete. Record the build ID from the output (format: `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`). The build typically takes 10-20 minutes.

    Check build status with: `eas build:view` or visit `https://expo.dev/accounts/[account]/projects/balance-tracker/builds`.

    **Important note:** Apple Developer Program enrollment must be active before this step. STATE.md notes: "start no later than Phase 11 to avoid blocking submission." If the Apple Developer Program is not yet enrolled, this step will fail at the provisioning profile creation step — in that case, document the failure and instruct the user to enroll at developer.apple.com/programs/enroll/ and re-run.

<task type="auto">
  <name>Task 3: App Store Connect metadata and EAS Submit</name>
  <files>
    BalanceTracker/eas.json
  </files>
  <action>
    After TestFlight verification passes (checkpoint approved), proceed with App Store metadata and submission.

    **Step 1: Update eas.json submit section**
    Fill in the real values in `BalanceTracker/eas.json` submit section:
    - `appleId`: the Apple ID email used for developer account
    - `ascAppId`: App Store Connect App ID (found in App Store Connect > App Information > Apple ID)
    - `appleTeamId`: Apple Developer Team ID (found at developer.apple.com > Account > Membership)

    These can also be set as environment variables to avoid storing in eas.json:
    - `EXPO_APPLE_ID`, `EXPO_ASC_APP_ID`, `EXPO_APPLE_TEAM_ID`

    **Step 2: Prepare App Store Connect app record (if not already created)**
    In App Store Connect (appstoreconnect.apple.com), the user must create a new app record:
    - Platform: iOS
    - Name: "Balance Tracker"
    - Bundle ID: com.balancetracker.app
    - SKU: balance-tracker-app (any unique string)

    Claude cannot automate App Store Connect web UI — document this as a required human action in SUMMARY.md and prompt the user via `echo` in the terminal.

    **Step 3: Prepare App Store metadata (document required fields)**
    Claude will output the required metadata fields as a formatted block so the user can copy-paste into App Store Connect:
    - **App Description**: "Balance Tracker is a personal finance app that helps you track income, expenses, debts, and assets in one place. Advanced mode adds client management, professional invoicing, and PDF export for freelancers."
    - **Keywords**: "finance tracker, expense tracker, income tracker, budget, invoicing, freelancer"
    - **Support URL**: "https://github.com/[username]/balance-tracker" (use the repo URL)
    - **Privacy Policy URL**: "https://balancetracker.app/privacy" — if no website exists, create a GitHub Gist with the privacy policy text and use the Gist raw URL
    - **Category**: Finance (Primary), Productivity (Secondary)
    - **Age Rating**: 4+
    - **Screenshots**: Required — 6.7" iPhone (iPhone 15 Pro Max) screenshots. Take from the TestFlight build on device or use Simulator. Minimum 3 screenshots.

    **Step 4: EAS Submit**
    Once the App Store Connect app record exists and metadata is complete, run:
    `eas submit --platform ios --latest`

    This submits the latest finished EAS build to App Store review.

    If `eas submit` fails due to missing credentials, alternative: in App Store Connect > TestFlight > [build] > click "Add to Review" and complete the submission wizard manually.
