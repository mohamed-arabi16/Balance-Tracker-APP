<objective>
Apply consistent iOS input styling and dark mode support across all four add/edit form screens.

Purpose: All four form screens (add-income, add-expense, add-debt, add-asset) were written independently and have hardcoded light-mode colors for inputs (#ffffff bg, #d1d5db border, #111827 text). In dark mode these are visually jarring — white inputs on a dark background. Additionally, the expense form has a red (#ef4444) save button which looks like an error state. This plan standardizes all forms to iOS system blue save buttons and adapts all input fields to dark mode.

Output: Four polished form screens with dark mode input support, consistent #007AFF primary actions, and a Cancel affordance.
</objective>
