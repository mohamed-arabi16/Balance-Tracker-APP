2. `export const useClients = ()` — useQuery with `queryKey: queryKeys.clients(user!.id)`, selects all columns, ordered by name ascending, `enabled: !!user`.

3. `export const useClient = (clientId: string)` — useQuery with `queryKey: [...queryKeys.clients(user!.id), clientId]`, selects single record by id AND user_id (RLS double-check), `.single()`, `enabled: !!user && !!clientId`.

4. `export const useAddClient = ()` — useMutation that inserts `{ ...payload, user_id: user!.id }`, returns created client, `onSuccess` invalidates `queryKeys.clients(user!.id)`.

5. `export const useUpdateClient = ()` — useMutation accepting `{ id: string } & ClientUpdate`, includes `updated_at: new Date().toISOString()` in the update payload (defensive: schema may not have a moddatetime trigger per STATE.md open question), `onSuccess` invalidates `queryKeys.clients(user!.id)` (prefix invalidation handles detail key automatically per queryKeys prefix-matching behavior).

6. `export const useDeleteClient = ()` — useMutation accepting clientId string, deletes where id AND user_id match, `onSuccess` invalidates `queryKeys.clients(user!.id)`.
