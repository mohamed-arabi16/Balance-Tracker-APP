    **add-debt.tsx** — Add/edit debt formSheet. Uses `useLocalSearchParams<{ id?: string }>()` for edit mode.

    When editing, find the debt from `useDebts()` cache by id.

    Form fields (useState, not react-hook-form):
    - Title: TextInput, required
    - Creditor: TextInput (who owes whom), required
    - Amount: TextInput, keyboardType="decimal-pad", required
    - Currency: `<Picker>` with USD, TRY
    - Type: `<Picker>` or toggle with values: short, long (short-term = under 30 days, long-term = over 30 days). Use i18n or plain labels "Short-term" / "Long-term".
    - Due Date: DateTimePicker, display="spinner", mode="date"
    - Is Receivable: Toggle/Switch — "I owe this debt" vs "Someone owes me". Use `<Pressable>` toggle. Default: false (I owe).
    - Status: `<Picker>` or toggle with values: pending, paid

    Submit (add): `useAddDebt().mutateAsync({ user_id: user.id, title, creditor, amount: parseFloat(amount), currency, type, due_date: dueDateValue.toISOString().split('T')[0], is_receivable, status })`

    Submit (edit): `useUpdateDebt().mutateAsync({ id, title, creditor, amount: parseFloat(amount), currency, type, due_date: dueDateValue?.toISOString().split('T')[0] ?? null, is_receivable, status, note: 'Updated' })`. CRITICAL: always pass `note` as 'Updated' for non-payment edits — this controls whether a payment history entry is semantically a payment or an edit. Do NOT pass `payment_date` for regular edits (only for payment flows).

    On success: `haptics.onSave()` then `router.back()`.

    Wrapper: `<FormScreen>` with Save button.

    ---

    **payment.tsx** — Make payment formSheet. Gets debt id from `useLocalSearchParams<{ id: string }>()`. Finds debt from `useDebts()` cache.

    Form fields (minimal):
    - Payment Amount: TextInput, keyboardType="decimal-pad", pre-filled with current debt amount, required
    - Payment Date: DateTimePicker, display="spinner", default = today

    On submit: `useUpdateDebt().mutateAsync({ id, title: debt.title, creditor: debt.creditor, amount: parseFloat(paymentAmount), currency: debt.currency, type: debt.type, due_date: debt.due_date, is_receivable: debt.is_receivable, status: parseFloat(paymentAmount) >= debt.amount ? 'paid' : 'pending', note: 'Payment', payment_date: paymentDate.toISOString() })`. Setting `note: 'Payment'` causes the updateDebt function to treat this as a payment in analytics/logging.

    On success: `haptics.onSave()` then `router.back()`.

    Wrapper: `<FormScreen>` with "Record Payment" button.

    Show the creditor name and current debt amount at the top of the form as context.

    Pitfall (from research, Pitfall 4): `useUpdateDebt` ALWAYS inserts a `debt_amount_history` row on every call. This is correct behavior for payment.tsx — we WANT a history entry. For add-debt.tsx edits, passing `note: 'Updated'` is fine semantically; the history will show "Updated amount" which is acceptable behavior.
