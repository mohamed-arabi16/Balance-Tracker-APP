    **i18n/index.ts — i18next initialization with RTL:**

    Build the i18n initialization following 07-RESEARCH.md Decision 2 exactly:

    1. Import `I18nManager` from `react-native`, `* as Localization` from `expo-localization`, `AsyncStorage` from `@react-native-async-storage/async-storage`
    2. Import `resources` from `./resources` (the translation strings extracted in Plan 01)
    3. Import `i18n` from `i18next` and `initReactI18next` from `react-i18next`

    4. Define `LANG_KEY = 'app_language'` constant for AsyncStorage

    5. Create `initI18n()` async function:
       - Read saved language from AsyncStorage: `const saved = await AsyncStorage.getItem(LANG_KEY)`
       - Detect device language: `const deviceLang = Localization.getLocales()[0]?.languageCode ?? 'en'`
       - Resolve language: `const language = saved ?? (deviceLang === 'ar' ? 'ar' : 'en')`
       - Apply RTL BEFORE any render: `I18nManager.allowRTL(language === 'ar')`
       - Initialize i18next: `await i18n.use(initReactI18next).init({ resources, lng: language, fallbackLng: 'en', interpolation: { escapeValue: false } })`
       - Return the resolved language

    6. Create `changeLanguage(lang: 'en' | 'ar')` async function:
       - Save to AsyncStorage: `await AsyncStorage.setItem(LANG_KEY, lang)`
       - Change i18next language: `await i18n.changeLanguage(lang)` (text updates immediately)
       - Return `true` if the RTL direction changed (i.e., new language RTL differs from current I18nManager.isRTL) — caller uses this to show the restart banner
       - Do NOT call `I18nManager.forceRTL()` here — that only applies on next cold start via `initI18n()`

    7. Export `initI18n`, `changeLanguage`, and the `LANG_KEY` constant

    **CONTEXT.md locked decisions honored:**
    - First launch detects iOS device language (Arabic devices get Arabic + RTL)
    - Language stored in AsyncStorage (not Supabase)
    - Language switch shows persistent banner (handled by caller, not this module)
    - RTL applied via I18nManager.allowRTL() at startup

    **ThemeContext.tsx — NativeWind system sync + manual override:**

    Build following 07-RESEARCH.md Decision 3:

    1. Import `useColorScheme` from `nativewind` and `AsyncStorage`
    2. Define `THEME_KEY = 'app_theme'` constant
    3. Define type: `type ThemePreference = 'light' | 'dark' | 'system'`

    4. Create `ThemeContext` with value: `{ theme: ThemePreference, isDark: boolean, setTheme: (t: ThemePreference) => void }`

    5. `ThemeProvider` component:
       - On mount, read saved preference from AsyncStorage
       - If saved preference is 'light' or 'dark', call NativeWind's `setColorScheme(preference)`
       - If saved preference is 'system' or not set, call `setColorScheme('system')` — NativeWind reads iOS Appearance API
       - `isDark` derived from `colorScheme === 'dark'` (from NativeWind's `useColorScheme()`)
       - `setTheme` function: saves to AsyncStorage, calls NativeWind's `setColorScheme()`

    6. Export `ThemeProvider` and `useTheme` hook (with context validation)
