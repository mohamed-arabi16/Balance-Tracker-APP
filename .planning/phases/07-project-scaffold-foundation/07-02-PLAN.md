    BalanceTracker/src/integrations/supabase/client.ts

  <action>
    Replace the placeholder Supabase client from Plan 01 with the production implementation.

    **CRITICAL ORDER: The first two imports MUST be in this exact order (P1 pitfall):**
    1. `import 'react-native-url-polyfill/auto'` — FIRST (URL polyfill)
    2. `import 'expo-sqlite/localStorage/install'` — SECOND (localStorage polyfill)

    Then create the Supabase client:
    ```typescript
    import { createClient } from '@supabase/supabase-js'
    import { AppState } from 'react-native'
    import type { Database } from './types'

    const supabaseUrl = process.env.EXPO_PUBLIC_SUPABASE_URL!
    const supabaseAnonKey = process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY!

    export const supabase = createClient<Database>(supabaseUrl, supabaseAnonKey, {
      auth: {
        storage: localStorage,      // provided by expo-sqlite polyfill
        autoRefreshToken: true,
        persistSession: true,
        detectSessionInUrl: false,  // no URL-based OAuth on mobile
      },
    })

    // Pause/resume auto-refresh based on app lifecycle
    AppState.addEventListener('change', (state) => {
      if (state === 'active') {
        supabase.auth.startAutoRefresh()
      } else {
        supabase.auth.stopAutoRefresh()
      }
    })
    ```

    **Do NOT use expo-secure-store** — it has a 2KB per-item size limit that Supabase sessions can exceed. The expo-sqlite localStorage polyfill is the officially documented approach.
  </action>

    **queryClient.ts:**
    Create `src/lib/queryClient.ts` with:
    - A `QueryClient` instance with default options: `staleTime: 1000 * 60 * 5` (5 min), `gcTime: 1000 * 60 * 30` (30 min), `retry: 1`
    - AppState-based focus manager: `focusManager.setEventListener()` that subscribes to AppState changes. When state changes to 'active', set focused to true; otherwise false. This makes React Query refetch stale queries when the user returns to the app.
    - Do NOT add NetInfo online manager — it requires a dev build (per 07-RESEARCH.md open question 3). React Query works fine without it; it just won't pause queries when offline. This can be added in Phase 8 when the dev build is required anyway.

    ```typescript
    import { QueryClient, focusManager } from '@tanstack/react-query'
    import { AppState, Platform } from 'react-native'
    import type { AppStateStatus } from 'react-native'

    function onAppStateChange(status: AppStateStatus) {
      if (Platform.OS !== 'web') {
        focusManager.setFocused(status === 'active')
      }
    }

    AppState.addEventListener('change', onAppStateChange)

    export const queryClient = new QueryClient({
      defaultOptions: {
        queries: {
          staleTime: 1000 * 60 * 5,
          gcTime: 1000 * 60 * 30,
          retry: 1,
        },
      },
    })
    ```
