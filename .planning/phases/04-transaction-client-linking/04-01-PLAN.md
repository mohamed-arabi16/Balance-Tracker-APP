Create `src/components/ClientCombobox.tsx` — a searchable client combobox using the Command+Popover pattern (same primitives already present in the project at `src/components/ui/command.tsx` and `src/components/ui/popover.tsx`).

Component signature:
```tsx
interface ClientComboboxProps {
  value: string | null;          // selected client_id or null
  onChange: (clientId: string | null) => void;
  placeholder?: string;          // defaults to t('income.form.placeholder.client')
  disabled?: boolean;
}

export function ClientCombobox({ value, onChange, placeholder, disabled }: ClientComboboxProps)
```

Implementation requirements:
1. Use `useClients()` from `@/hooks/useClients` to load clients (Client[] with `id` and `name`).
2. Use `useState` for `open` (popover open state) and `search` (filter string).
3. Trigger: a `<Button variant="outline">` (full width, `w-full justify-start text-left font-normal`) showing either the selected client name or the placeholder text (`cn("text-muted-foreground", ...)` when no client selected). Right side: when a client is selected, show an `<X className="h-4 w-4 opacity-50 hover:opacity-100" />` icon button that calls `onChange(null)` and `e.stopPropagation()` to clear without opening the popover. Use a `<ChevronsUpDown className="h-4 w-4 opacity-50" />` icon on the left when no client is selected.
4. Wrap in `<Popover open={open} onOpenChange={setOpen} modal={true}>` — `modal={true}` is required because this will be inside a Dialog (same pattern used by the date pickers in Income.tsx and Expenses.tsx).
5. `<PopoverContent className="w-[--radix-popover-trigger-width] p-0">` — match trigger width.
6. Inside PopoverContent: `<Command>` → `<CommandInput placeholder={t('clients.searchPlaceholder')} value={search} onValueChange={setSearch} />` → `<CommandList>` → `<CommandEmpty>` with message and link.
7. `<CommandEmpty>`: render `t('income.form.noClients')` text plus a `<Link to="/clients/new" className="text-primary underline">` link showing `t('income.form.addClient')`. Import `Link` from `react-router-dom`.
8. Filter clients by `search` string (case-insensitive match on `client.name`). Render each as `<CommandItem key={client.id} value={client.name} onSelect={() => { onChange(client.id); setOpen(false); setSearch(''); }}>` with `<Check className="mr-2 h-4 w-4 {value === client.id ? 'opacity-100' : 'opacity-0'}" />` for selection indicator.
9. Imports from lucide-react: `Check`, `ChevronsUpDown`, `X`.
10. Use `useTranslation` from `react-i18next` for all visible strings.
11. Handle loading state: if `useClients()` returns `isLoading: true`, show `<CommandEmpty>` with loading text; use clients data once available (default to empty array `[]` if undefined).
