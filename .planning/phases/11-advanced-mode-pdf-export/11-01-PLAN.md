**Step B — Create app/(tabs)/_layout.tsx with conditional Clients and Invoices tabs:**

Phase 8 will build the full tab bar. For Phase 11 planning, this file may already exist from Phase 8. If it exists, MODIFY it to add the conditional Clients and Invoices tabs. If it does not exist yet (Phase 11 executing before Phase 8), CREATE a minimal tab layout.

The tab bar from the roadmap has 5 simple-mode tabs: Dashboard (index), Transactions, Debts, Assets, More/Settings. Advanced mode adds Clients (conditional) and Invoices accessible from Clients flow.

Create or modify `app/(tabs)/_layout.tsx`:
```tsx
import { Tabs } from 'expo-router';
import { useTranslation } from 'react-i18next';
import { useMode } from '@/contexts/ModeContext';

export default function TabLayout() {
  const { t } = useTranslation();
  const { isAdvanced } = useMode();

  return (
    <Tabs screenOptions={{ headerShown: false }}>
      <Tabs.Screen
        name="index"
        options={{ title: t('nav.dashboard', 'Dashboard') }}
      />
      <Tabs.Screen
        name="transactions"
        options={{ title: t('nav.transactions', 'Transactions') }}
      />
      <Tabs.Screen
        name="debts"
        options={{ title: t('nav.debts', 'Debts') }}
      />
      <Tabs.Screen
        name="assets"
        options={{ title: t('nav.assets', 'Assets') }}
      />
      <Tabs.Screen
        name="settings"
        options={{ title: t('nav.settings', 'Settings') }}
      />
      {/* Advanced mode tabs — hidden via href: null when not in advanced mode */}
      <Tabs.Screen
        name="clients"
        options={{
          title: t('nav.clients', 'Clients'),
          href: isAdvanced ? undefined : null,
        }}
      />
      <Tabs.Screen
        name="invoices"
        options={{
          title: t('nav.invoices', 'Invoices'),
          href: isAdvanced ? undefined : null,
        }}
      />
    </Tabs>
  );
}
```

Note: `href: undefined` (not `/clients`) means "use default tab routing" and shows the tab. `href: null` hides it.

Also add a minimal `app/(tabs)/clients/index.tsx` and `app/(tabs)/invoices/index.tsx` stub screens (just a SafeScreen with a placeholder Text) so Expo Router doesn't error on the Tabs.Screen name references before Plans 02 and 03 build the real screens.
