## Decisions Made

- **expo-router typed routes:** The `.expo/types/router.d.ts` is gitignored (auto-generated by dev server). New `(auth)` route paths cause TypeScript errors when used in `<Link href="/(auth)/sign-in">`. Used `as any` type assertions as the pragmatic fix — routes will be correctly typed after first `expo start` regenerates the file.
- **queryClient.clear() placement:** Only called in explicit `signOut()` function, not inside `onAuthStateChange`. This avoids wiping cache on token auto-refresh events which also trigger the auth state change.
- **No redirectTo in resetPassword:** Phase 12 will add deep link handling. For now Supabase sends the default browser link.
