2. Replace the ENTIRE contents of `BalanceTracker/src/contexts/AuthContext.tsx` with a real Supabase auth implementation:

**Interface:**
```typescript
interface AuthContextType {
  user: User | null;        // from @supabase/supabase-js
  session: Session | null;  // from @supabase/supabase-js
  isLoading: boolean;
  signIn: (email: string, password: string) => Promise<{ error: Error | null }>;
  signUp: (email: string, password: string) => Promise<{ error: Error | null }>;
  signOut: () => Promise<void>;
  resetPassword: (email: string) => Promise<{ error: Error | null }>;
}
```

**AuthProvider behavior:**
- `isLoading` starts as `true`
- On mount `useEffect`: call `supabase.auth.getSession()` to hydrate from persisted session (expo-sqlite localStorage polyfill). Set `session` and `isLoading = false` when resolved.
- Subscribe to `supabase.auth.onAuthStateChange((_event, session) => { setSession(session); setIsLoading(false); })`. Return cleanup: `subscription.unsubscribe()`.
- `signIn`: calls `supabase.auth.signInWithPassword({ email, password })`, returns `{ error }`.
- `signUp`: calls `supabase.auth.signUp({ email, password })`, returns `{ error }`.
- `signOut`: calls `supabase.auth.signOut()`, then `queryClient.clear()` from `@/lib/queryClient` to wipe cached user data. Do NOT call `queryClient.clear()` inside `onAuthStateChange` — only in the explicit `signOut()` function.
- `resetPassword`: calls `supabase.auth.resetPasswordForEmail(email)` (no `redirectTo` — Supabase handles via browser for now, Phase 12 adds deep link), returns `{ error }`.
- `user` derived as `session?.user ?? null`.

**Exports:** `AuthProvider`, `useAuth` (with context-undefined guard).

Import `Session` and `User` types from `@supabase/supabase-js`. Import `supabase` from `@/integrations/supabase/client`. Import `queryClient` from `@/lib/queryClient`.
