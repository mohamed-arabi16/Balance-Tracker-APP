<objective>
Fix 4 critical UI regressions found in screenshots after Phase 13 execution. The app is currently unusable: Settings is broken colored strips, Dashboard cards are invisible (white on white), form pickers show no value, and list rows are dark in light mode.

All issues stem from NativeWind class failures and missing page background colors.
</objective>

<tasks>

<task id="1" type="code">
  <name>Fix Settings screen — icon badge NativeWind failure</name>
  <problem>
    In settings.tsx, the SettingsRow icon badge uses NativeWind className="w-8 h-8 rounded-lg items-center justify-center mr-3 flex-shrink-0" with style={{ backgroundColor: iconBg }}.
    NativeWind w-8/h-8 is not constraining the View width/height, so backgroundColor fills the full row width → giant colored stripe.
  </problem>
  <fix>
    Replace NativeWind w-8/h-8 on the icon badge with StyleSheet-based explicit width/height.
    In settings.tsx SettingsRow, change the icon container from:
      className="w-8 h-8 rounded-lg items-center justify-center mr-3 flex-shrink-0"
    To use inline style:
      style={{ width: 32, height: 32, borderRadius: 8, alignItems: 'center', justifyContent: 'center', marginRight: 12, flexShrink: 0, backgroundColor: iconBg }}
    Remove the separate style={{ backgroundColor: iconBg }} prop since it's now in the combined style object.
  </fix>
  <files>app/(tabs)/settings.tsx — SettingsRow component, icon badge View</files>
</task>

<task id="2" type="code">
  <name>Fix Dashboard — cards invisible on white background</name>
  <problem>
    Dashboard ScrollView contentContainerStyle has padding: 16 but no background color.
    All cards use bg-white which matches the white page background — cards are invisible.
    SafeScreen background is white. No grouped background separating page from cards.
  </problem>
  <fix>
    Add backgroundColor: '#F2F2F7' to the ScrollView contentContainerStyle in app/(tabs)/index.tsx.
    Also add the same background to SafeScreen in the dashboard — wrap the ScrollView:
      <SafeScreen edges={['top']} style={{ backgroundColor: '#F2F2F7' }}>
    Cards (bg-white) will then pop against the gray grouped background — standard iOS pattern.
    For dark mode, use COLORS.groupedBg.dark ('#1C1C1E') in dark mode.
    Since SafeScreen and ScrollView use StyleSheet, use useColorScheme from nativewind:
      const { colorScheme } = useColorScheme();
      const pageBg = colorScheme === 'dark' ? '#1C1C1E' : '#F2F2F7';
  </fix>
  <files>app/(tabs)/index.tsx — DashboardScreen, ScrollView contentContainerStyle + SafeScreen</files>
</task>

<task id="3" type="code">
  <name>Fix form pickers — Currency and Category values invisible</name>
  <problem>
    After changing pickerWrapper to backgroundColor: '#F2F2F7', the Picker component inside
    renders with its own default background (white or transparent) and the Picker.Item labels
    may not be visible because the Picker ignores the wrapper background on iOS.
    The picker wrapper shows gray fill but the Picker itself renders transparently over it,
    and on iOS the Picker component in "spinner" style (height 44) collapses and shows nothing.
  </problem>
  <fix>
    For iOS, the native Picker with style={{ height: 44 }} in a wrapper is problematic.
    Replace the Picker components in all 4 forms with Pressable buttons that show current value
    and open an ActionSheet or a modal picker — same pattern already used for the client picker.

    Specifically for Currency and Category fields in all 4 forms:
    Replace <View style={pickerWrapper}><Picker .../></View> with a Pressable row that:
    - Shows the current value as text (style: input)
    - Has a SymbolView chevron.up.chevron.down or chevron.right on right side
    - Tapping opens a Modal (same pattern as clientButton/clientModalVisible already in income/expense forms)

    This is more reliable than the native Picker on iOS and looks much better.
    Reuse the existing modal + FlatList pattern from the client picker already in the forms.
  </fix>
  <files>
    app/(tabs)/transactions/add-income.tsx
    app/(tabs)/transactions/add-expense.tsx
    app/(tabs)/debts/add-debt.tsx
    app/(tabs)/assets/add-asset.tsx
  </files>
</task>
