<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Verify cascade delete behavior for both FK types</name>
  <action>
Run two SQL test sequences in the Supabase Dashboard SQL Editor to verify the FK cascade behaviors. These queries create test data and clean up after themselves.

TEST A — Verify ON DELETE RESTRICT on invoices.client_id:

Run this SQL:
```sql
-- Create test data
INSERT INTO clients (user_id, name) VALUES (auth.uid(), 'Test Client RESTRICT');
INSERT INTO invoices (user_id, client_id, invoice_number, issue_date)
  SELECT auth.uid(), id, 'TEST-RESTRICT-001', CURRENT_DATE
  FROM clients WHERE name = 'Test Client RESTRICT' AND user_id = auth.uid();

-- Attempt to delete client (SHOULD FAIL with FK violation)
DELETE FROM clients WHERE name = 'Test Client RESTRICT' AND user_id = auth.uid();
```

Expected: ERROR — "update or delete on table 'clients' violates foreign key constraint 'invoices_client_id_fkey'"
If DELETE succeeds without error: the cascade behavior is WRONG — stop and report.

Cleanup (run after confirming the error):
```sql
DELETE FROM invoices WHERE invoice_number = 'TEST-RESTRICT-001' AND user_id = auth.uid();
DELETE FROM clients WHERE name = 'Test Client RESTRICT' AND user_id = auth.uid();
```

TEST B — Verify ON DELETE SET NULL on incomes.client_id:

Run this SQL:
```sql
-- Create test data
INSERT INTO clients (user_id, name) VALUES (auth.uid(), 'Test Client SET NULL');
INSERT INTO incomes (user_id, title, amount, currency, date, category, client_id)
  SELECT auth.uid(), 'Test income SET NULL', 100, 'USD', CURRENT_DATE, 'test',
         (SELECT id FROM clients WHERE name = 'Test Client SET NULL' AND user_id = auth.uid());

-- Delete the client (SHOULD SUCCEED)
DELETE FROM clients WHERE name = 'Test Client SET NULL' AND user_id = auth.uid();

-- Verify the income row still exists with client_id = NULL
SELECT id, title, client_id FROM incomes WHERE title = 'Test income SET NULL' AND user_id = auth.uid();
```

Expected:
- DELETE succeeds with no error
- SELECT returns 1 row with client_id = NULL

If DELETE fails with FK error: ON DELETE SET NULL not set — WRONG. Stop and report.
If SELECT returns 0 rows: ON DELETE CASCADE was applied instead — income was deleted — WRONG. Stop and report.

Cleanup:
```sql
DELETE FROM incomes WHERE title = 'Test income SET NULL' AND user_id = auth.uid();
```
