# Requirements: Balance Tracker — Advanced Mode

**Defined:** 2026-02-23
**Core Value:** Anyone can track their money simply — and freelancers can manage their business without switching apps.

## v1 Requirements

### Mode System

- [x] **MODE-01**: User can toggle between Simple and Advanced mode with a single button click from anywhere in the app
- [x] **MODE-02**: User can set a default mode preference (Simple or Advanced) in the Settings page
- [x] **MODE-03**: Accessing advanced routes (/clients, /invoices, /advanced) while in Simple mode redirects user to dashboard
- [x] **MODE-04**: App opens in the user's last active mode on page refresh (mode persisted to user_settings)

### Client Management

- [x] **CLNT-01**: User can create a client with name, email, phone, company, and notes fields
- [x] **CLNT-02**: User can edit an existing client's details
- [x] **CLNT-03**: User can view a list of all clients with text search
- [x] **CLNT-04**: User can view a client detail page showing all invoices and linked transactions for that client

### Invoices

- [x] **INV-01**: User can create an invoice linked to a client, with one or more line items (description, quantity, rate, calculated total)
- [x] **INV-02**: Invoice is auto-assigned a unique sequential invoice number on creation
- [x] **INV-03**: User can transition invoice status: Draft → Sent → Paid; overdue status is auto-derived when due_date has passed and status is still Sent
- [x] **INV-04**: Draft invoices can be fully edited; Sent and Paid invoices are read-only (locked from editing)
- [x] **INV-05**: User can export any invoice as a PDF file (client-side generation, lazy-loaded PDF library)
- [x] **INV-06**: When marking an invoice as Paid, app offers an optional prompt to create a matching income entry

### Transaction-Client Linking

- [x] **TXN-01**: User can optionally select a client when creating a new income entry
- [x] **TXN-02**: User can optionally select a client when creating a new expense entry

### Advanced Dashboard

- [ ] **DASH-01**: Advanced mode has its own dashboard showing revenue per client (total paid invoices per client)
- [ ] **DASH-02**: Advanced dashboard shows outstanding invoices panel (all Sent and Overdue invoices with amounts owed)

### Internationalization

- [x] **I18N-01**: All Advanced Mode UI strings have Arabic translations and correct RTL layout, matching the quality standard of the existing app

## v2 Requirements

### Client Management

- **CLNT-V2-01**: User can archive (soft-delete) a client — removed from active list without losing historical data
- **CLNT-V2-02**: User can retroactively tag existing income entries to a client

### Dashboard & Analytics

- **DASH-V2-01**: Advanced dashboard shows monthly profit trend chart (income vs expenses over time)
- **DASH-V2-02**: P&L summary for a selected date range (total income minus total expenses)
- **DASH-V2-03**: P&L breakdown per client (profit/loss attributed to each client)

### Invoices

- **INV-V2-01**: Recurring invoice templates (duplicate existing invoice as new draft)
- **INV-V2-02**: Client portal — shareable link for client to view invoice

## Out of Scope

| Feature | Reason |
|---------|--------|
| Email sending of invoices | User exports PDF and sends manually — avoids email infrastructure complexity |
| Tax calculation / withholding | Too jurisdiction-specific; wrong to implement without legal accuracy |
| Native mobile app | PWA covers mobile use for now |
| Multi-user / team accounts | Requires fundamentally different auth and RLS model |
| Subscription / recurring billing | Requires external payment processor integration |
| Time tracking | Full sub-feature; high complexity with low overlap with current user base |
| Retroactive tagging of existing transactions | V2 — client field on new transactions is sufficient for v1 launch |
